Group Tips
Give the group a clear, descriptive name like "Event Managers" or "Viewers".
Permissions follow the pattern action_model — e.g. add_event.
You can always edit the group later to add or remove permissions.
Keep groups minimal — assign only the permissions each role truly needs.
How It Works
1
Name the group — choose a role-based name that reflects the level of access.
2
Pick permissions — use the filter box to quickly find what you need.
3
Create the group — then assign users to it from the User Management page.
